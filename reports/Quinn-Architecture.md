# Quinn 投資分析師 — 架構說明

## 🎭 Quinn 是什麼？

**Quinn 是小c 的一個「角色」**，由老大 (您) 召喚啟動。

不是獨立機器人、不是 cron job、不是另一個 agent。

當您在 Discord 對話中提到「Quinn」或投資相關問題時，**我 (小c) 切換到 Quinn 角色**，以 14 年投資分析師身份回應。

## 🏗️ 正確架構 (修正後)

```
                  ┌─────────────────────┐
                  │   小c (Quinn 角色)    │ ← 主控核心 (LLM)
                  │   ┌───────────────┐   │
                  │   │ 整合所有資訊    │   │
                  │   │ 產出分析報告    │   │
                  │   │ 風格/語境/學習  │   │
                  │   └───────────────┘   │
                  └──────────┬──────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
   ┌─────────┐         ┌──────────┐         ┌──────────┐
   │ 資料層  │         │ 計算層    │         │ 呈現層    │
   │ TWSE API │         │ Python    │         │ Web/SQLite│
   │ TPEx API │         │ Scripts  │         │ Discord  │
   │ News     │         │ MA/RSI   │         │          │
   └─────────┘         └──────────┘         └──────────┘
```

## ❌ 不正確的舊架構 (Quinn 已修正)

```
❌ cron 觸發 → 死指令 prompt → 另一個 LLM 產生報告
   (Quinn 寫 prompt 取代自己，這是錯誤的)
```

問題：
- 報告沒有靈魂 (風格不統一)
- 沒有跨上下文整合 (不記得您之前的對話)
- 無法即時互動 (您不能追問)
- 浪費 LLM (prompt 寫得不好的話品質差)

## 🎯 各層職責分工

### 1️⃣ 資料層 (cron 負責 - 純資料抓取)

| 時程 | 任務 | 工具 |
|------|------|------|
| 08:55 | 啟動盤中監控守護 | `intraday_monitor.py --daemon` |
| 09:00–13:30 | 抓即時報價、觸發 Buy Zone | TWSE MIS API + Python |
| 13:30 | 抓 11 支收盤價 | `fetch_prices.py` |
| 14:00 | DB 同步 | `db_init.py` |

### 2️⃣ 計算層 (Python 處理 - 不需要 LLM)

- MA/RSI 計算 (5 筆以上歷史資料後)
- Buy Zone 觸發判定
- 觸發訊號冷卻 (60 分鐘內同訊號不重發)
- Discord 訊息發送 (`hermes send`，已經是純指令)
- 週報 MD 檔案生成 (有結構的 Markdown)

### 3️⃣ LLM 層 (Quinn 自己 - 整合分析)

**Quinn (小c) 主動觸發的情境**：
1. **盤後分析** — 老大打開 Discord / 網頁 / 下次對話時，Quinn 自動判斷當天已收盤，主動產生分析
2. **每週選股會議** — 週日老大開啟對話時，Quinn 自動召開會議
3. **突發事件** — 觸發訊號 (Buy Zone 觸發) 透過 Discord 通知到老大，Quinn 在老大回應時深度分析
4. **新標的研究** — 老大指定新標的或掃描器推薦時，Quinn 研究後產出完整報告

### 4️⃣ 呈現層 (Web + Discord 通知)

- **Web 頁面** (`localhost:5050`) — 隨時查詢 11 支監控狀態
- **Discord 通知** — 觸發訊號即時推送
- **GitHub Repo** (`ttcccat-tech/tw-stock-research`) — 完整研究報告歷史

## 📂 容器內 cron vs Hermes cron 差異

### 容器 cron (`crontab`)
- ✅ **保留**：純資料抓取 (收盤價、DB 同步、守護啟動)
- ❌ **移除**：自動分析報告 (改由 Quinn LLM 觸發)

### Hermes cron (hermes-agent)
- ✅ **保留**：每日收盤價抓取 (純腳本)
- ✅ **保留**：盤中監控守護啟動 (純腳本)
- ❌ **移除**：「每週選股會議」 (改由 Quinn LLM 觸發)
